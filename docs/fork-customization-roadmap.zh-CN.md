# OpenHands Fork 深度改造清单（中文）

> 目标：把 OpenHands 主项目与 `openhands-ui` 子项目从“上游开源仓库”改造成“可持续维护的个人/团队产品仓库”。
> 说明：每个项目给出不少于 50 条建议，并标记本轮是否已实施。

## C. 本轮落地范围说明

本轮优先完成“低风险、可立即使用、对品牌化最关键”的改动：

- 双 README 中文重写与结构升级
- 自有 Logo 与品牌元数据模板
- 自定义协议文件
- Docker 与配置模板参数化

其余涉及大规模重命名、依赖升级回归、CI 重构的项，建议分两到三轮推进，并在每轮后跑完整回归测试。
